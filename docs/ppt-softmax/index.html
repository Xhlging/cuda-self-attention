THE FORMULA

Softmax 是什么？

softmax(xi)
exi
Σ exj
直观理解

把一组任意实数 → 转成一组概率。

大的数 → 高概率 (接近 1)
小的数 → 低概率 (接近 0)
全部加起来 = 1

例: softmax([2, 1, -1])
= [0.67, 0.24, 0.09]

THE PROBLEM

为什么要用 GPU 算 Softmax？

场景

大语言模型（如 ChatGPT）每一层、每个 token 都要算一次 Softmax。

例如: 32,768 个 token
每个 token 有 4,096 维
= 每次推理要算 32,768 次 Softmax

GPU 方案

每个线程独立算一行 Softmax。
几千行 → 几千个线程同时算。

问题: 同一行内，怎么让
32 个线程协作加速？

NAIVE · 04
04 / 10
VERSION 1

Naive Softmax: 每个线程单干

// 线程 tid 独立计算第 row 行的 softmax
// 第一遍: 找最大值
max = x[0]
for i=1..D-1: max = max(max, x[i])

// 第二遍: 算 exp + 求和
sum = 0
for i=0..D-1:
  y[i] = exp(x[i] - max)
  sum += y[i]

// 第三遍: 归一化
for i=0..D-1: y[i] /= sum
问题在哪？

每个线程串行遍历 D 个元素三次。

如果 D=1024，一个线程要遍历 3072 次。
线程之间完全没有协作。

Grid: [rows/256]
Block: [256]
每线程: 串行三遍
WARP · 05
05 / 10
KEY CONCEPT

Warp: GPU 的最小调度单位

什么是 Warp？

32 条线程编成一组 = 1 个 Warp
GPU 以 Warp 为单位调度执行

线程 0 ─┬─ Warp 0
线程 1 ─┤
...      │
线程 31─┘
Warp 的特殊能力

__shfl_down_sync: 同 Warp 内的线程可以直接交换数据——不需要共享内存！

线程 i 收到线程 i+offset 的值
→ 蝴蝶归约 → log₂(32)=5 步
→ 32 个值合并成 1 个

REDUCE · 07
07 / 10
REDUCTION + BROADCAST

归约产生在 lane 0，广播分发给所有人

Step A: 蝴蝶归约 (Reduce)
// 每轮折半，下家把自己的值传给上家
for (int offset = 16; offset > 0; offset /= 2)
  sum_val += __shfl_down_sync(
    0xFFFFFFFF, sum_val, offset);
offset=16
lane 0←16, 1←17, ..., 15←31 → 剩 16 个
offset=8
lane 0←8, 1←9, ..., 7←15 → 剩 8 个
...
offset=1
lane 0←1 → 只在 lane 0
Step B: 广播 (Broadcast)
// 把 lane 0 的值复制给所有 32 条线程
sum_val = __shfl_sync(
  0xFFFFFFFF, sum_val, 0);
__shfl_sync(mask, val, src_lane)
src_lane=0 → 线程 0 的值分发给 Warp 内所有人
两步缺一不可

归约 把 32 个值合并为 1 个（落在 lane 0）
广播 把这 1 个值复制给人人（所有 lane 都拿到）
少一步 → 只有 lane 0 是对的，其他 31 个线程得到错误结果

WARP SOFTMAX KERNEL

完整代码: 35 行实现 Warp Softmax

// Grid: [rows], Block: [32] — 一个 Warp 处理一行 template <int D> __global__ void softmax_warp(const float* x, float* y, int rows, int stride) { int row = blockIdx.x; // 一行一个 Block int lane = threadIdx.x; // 线程在 Warp 内的编号 (0-31) if (row >= rows) return; x += row * stride; y += row * stride; // === Step 1: 每个线程处理 D/32 个元素，找局部最大值 === float max_val = -INFINITY; for (int i = lane; i < D; i += 32) max_val = fmaxf(max_val, x[i]); // === Step 2: 蝴蝶归约，32 个局部 max → 1 个全局 max === for (int offset = 16; offset > 0; offset /= 2) max_val = fmaxf(max_val, __shfl_down_sync(0xFFFFFFFF, max_val, offset)); max_val = __shfl_sync(0xFFFFFFFF, max_val, 0); // ← 广播！ // === Step 3: 算 exp + warp 归约求和 === float sum_val = 0; for (int i = lane; i < D; i += 32) { float e = __expf(x[i] - max_val); y[i] = e; sum_val += e; } for (int offset = 16; offset > 0; offset /= 2) sum_val += __shfl_down_sync(0xFFFFFFFF, sum_val, offset); sum_val = __shfl_sync(0xFFFFFFFF, sum_val, 0); // ← 广播！ // === Step 4: 归一化 === float inv_sum = 1.0f / sum_val; for (int i = lane; i < D; i += 32) y[i] *= inv_sum; }
